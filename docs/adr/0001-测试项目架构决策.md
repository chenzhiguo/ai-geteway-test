# ADR-0001: 测试项目架构决策

## 状态

已接受

## 日期

2026-05-24

## 背景

需要为AI Gateway创建一个自动化测试项目，用于测试网关的所有接口类型和性能特性。

## 决策

### 1. 测试框架选择

**选择**：pytest

**理由**：
- Python最流行的测试框架
- 插件生态系统丰富
- 支持参数化、fixture等高级功能
- 与Allure报告集成良好

**替代方案**：
- unittest：标准库，但功能相对简单
- nose2：unittest的扩展，但社区活跃度较低

### 2. 依赖管理工具选择

**选择**：Poetry

**理由**：
- 现代化的Python依赖管理工具
- 支持锁文件，确保依赖版本一致
- 内置虚拟环境管理
- 配置文件格式清晰

**替代方案**：
- pip + requirements.txt：传统方式，但缺乏锁文件支持
- pipenv：结合pip和virtualenv，但性能较差

### 3. 测试层次划分

**选择**：E2E测试 + 性能测试

**理由**：
- E2E测试覆盖完整的请求流程
- 性能测试验证网关的吞吐量、延迟和并发能力
- 单元测试在网关项目内部完成

**替代方案**：
- 仅E2E测试：无法验证性能特性
- 全层次测试：增加复杂度，收益有限

### 4. Mock服务器实现

**选择**：Flask实现的Mock服务器

**理由**：
- Flask轻量级，易于实现
- 支持流式响应（SSE）
- 可以模拟各种LLM提供商的API

**替代方案**：
- 使用WireMock：Java实现，需要额外运行环境
- 使用responses库：仅支持requests库，不够灵活

### 5. 测试报告工具

**选择**：Allure + pytest-html + 终端输出

**理由**：
- Allure提供专业的测试报告，支持趋势、分类等
- pytest-html提供简单的HTML报告
- 终端输出方便快速查看结果

**替代方案**：
- 仅终端输出：信息不够丰富
- 仅Allure：配置复杂

### 6. 配置管理方式

**选择**：YAML配置文件 + 环境变量替换

**理由**：
- YAML格式清晰，易于阅读
- 支持环境变量替换，便于不同环境配置
- 支持配置继承和覆盖

**替代方案**：
- 仅环境变量：配置项多时不够清晰
- JSON配置文件：不支持注释

### 7. 项目组织结构

**选择**：按测试层次组织（e2e/、performance/）

**理由**：
- 结构清晰，易于理解
- 便于按类型运行测试
- 符合测试项目的常见组织方式

**替代方案**：
- 按接口类型组织：会导致代码重复
- 按功能模块组织：与网关结构耦合过紧

### 8. 性能测试方法

**选择**：自定义性能测试 + Locust

**理由**：
- 自定义测试可以精确控制测试场景
- Locust支持分布式负载测试
- 可以生成详细的性能报告

**替代方案**：
- 仅使用Locust：配置复杂
- 仅自定义测试：缺乏分布式能力

### 9. 日志记录方式

**选择**：Python logging模块 + 结构化日志

**理由**：
- Python标准库，无需额外依赖
- 支持多种日志级别和输出格式
- 可以输出到控制台和文件

**替代方案**：
- 使用loguru：第三方库，功能强大但增加依赖
- 使用structlog：结构化日志，但配置复杂

### 10. CI/CD集成方式

**选择**：GitHub Actions + 质量门禁

**理由**：
- GitHub Actions免费且易于配置
- 支持测试覆盖率检查
- 支持测试结果通知

**替代方案**：
- GitLab CI：需要GitLab环境
- Jenkins：配置复杂

## 后果

### 正面影响

1. **易于维护**：清晰的项目结构和文档
2. **易于扩展**：模块化设计，便于添加新测试
3. **全面覆盖**：覆盖所有接口类型和性能特性
4. **专业报告**：支持多种测试报告格式

### 负面影响

1. **学习成本**：需要学习pytest、Poetry等工具
2. **依赖管理**：需要维护多个依赖
3. **环境要求**：需要Python 3.10+环境

### 风险和缓解措施

1. **风险**：依赖版本冲突
   **缓解**：使用Poetry锁文件，定期更新依赖

2. **风险**：测试环境不稳定
   **缓解**：使用Mock服务器，减少对外部依赖

3. **风险**：测试用例维护成本高
   **缓解**：编写清晰的文档，使用fixture减少重复代码
